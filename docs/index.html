Getting Started with MongoDiff

A step-by-step tutorial for comparing MongoDB collections and Java lists, visualizing differences, and exporting reports.

Java 17+ Spring Boot 3.2 MongoDB React 18 MIT License
1 Overview

What MongoDiff Does

MongoDiff is a regression testing and data validation tool. It compares two data sources field-by-field and shows you exactly what changed.

⚔ Two Comparison Modes

Compare MongoDB collections directly or Java lists in-memory — no database required.

⚡ Merge-Join Algorithm

Efficient sorted-stream comparison handles millions of records in O(n log n) time.

🔍 Field-Level Diffs

Identifies exact attributes that differ between matched records across four break types.

📊 React Web UI

Dark IDE-inspired interface with real-time progress, filtering, sorting, and drill-down views.

Every comparison classifies each record into one of four break types:

match difference onlyOnA onlyOnB
2 Installation

Prerequisites & Build

You need Java 17+ and Maven 3.6+ installed. MongoDB is optional (only for MongoDB-mode comparisons).

bash
# Clone the repository
git clone https://github.com/drompincen/mongodiff.git
cd mongodiff

# Build the project
mvn clean package

# Run the application
java -jar target/comparison-app-1.0.0.jar
✅

No MongoDB needed! MongoDiff starts without a running MongoDB instance. The in-memory comparison mode works out of the box.

Once running, open http://localhost:8080 in your browser.

4 UI Tour

Navigating the Interface

MongoDiff has a dark, IDE-inspired web UI with four main screens. Click any screenshot to enlarge.

4.1 — Start Session Screen

The landing page where you create a new comparison session. Enter a session name, optionally paste a CSV of IDs to scope the comparison, then choose how to load data.

  • Start Session — begin with your custom configuration
  • Load Demo (80 IDs) — quick demo with 80 records
  • Load Sample (MongoDB) — load 215 accounts into MongoDB and compare
  • Load Mongo Mem — same 215 accounts compared entirely in-memory

4.2 — Results Grid (Comparison Dashboard)

After loading data, the dashboard displays:

  • Stat cards at the top — Total IDs, Baseline loaded, RC loaded, Compared count, Avg Match %, Categories
  • Progress bars — real-time loading and comparison progress
  • Data grid — sortable and filterable table with ID, Name, Category, match/break bar, Status, Duration, and a View button for completed records
  • Excel export — download a color-coded .xlsx report

4.3 — Break Detail View (Drill-Down)

Click "View" on any completed record to drill into the field-by-field comparison.

  • FIELD column lists every compared attribute
  • BASELINE (A) shows the value from collection A (highlighted yellow)
  • RC (B) shows the value from collection B (highlighted green)
  • TYPE badge indicates difference for each mismatched field
  • A summary bar at the top shows overall match percentage and break count

4.4 — Active Sessions Sidebar

Click the "Sessions" button in the top-right corner to open the sidebar. It shows all active comparison sessions across your team, with their status and progress.

5 API Reference

REST Endpoints

MongoDiff exposes a simple REST API for triggering comparisons and retrieving results.

EndpointMethodDescription
/api/sample/load POST Load 215 sample accounts into MongoDB, compare, and return results
/api/sample/load-mem POST Same comparison done entirely in-memory (no MongoDB needed)
/api/sample/status GET Get the status and results of the last comparison
/api/sample/breaks/{id} GET Get field-level breaks for a specific record ID

6 Code Usage

Programmatic Integration

Use the comparison engine directly in your Java code for automated regression testing.

Compare MongoDB Collections

java
@Autowired GenericComparisonService comparisonService;

comparisonService.compareCollections(
    Account.class,
    "accountBaseline",          // collection A
    "accountRC",                // collection B
    "accountId",                // key attribute
    List.of("balance", "accountType", "riskLevel"),
    "comparisonBreaks"          // output collection
);

Compare Java Lists (No MongoDB)

java
GenericComparisonService service = new GenericComparisonService();

ListComparisonResult<Account> result = service.compareLists(
    baselineAccounts,           // list A
    rcAccounts,                 // list B
    "accountId",                // key attribute
    List.of("balance", "accountType", "riskLevel")
);

System.out.println(result);
// List Comparison Summary:
//   Items Processed from List A: 205
//   Items Processed from List B: 210
//   Keys Only in List A: 5
//   Keys Only in List B: 10
//   Common Keys Found: 200
//     - Fully Matched Keys: 170
//     - Keys with Attribute Mismatches: 30
//   Total Individual Attribute Differences: 30

Generate Excel Report

java
@Autowired ExcelReportService excelService;

XSSFWorkbook workbook = excelService.generateExcelReport(
    Account.class,
    "accountId",
    "accountBaseline",
    "accountRC",
    "comparisonBreaks"
);

try (FileOutputStream out = new FileOutputStream("report.xlsx")) {
    workbook.write(out);
}

differenceSide-by-side values with yellow (A) and green (B) highlighting

Comparison Engine

The GenericComparisonService uses a merge-join algorithm on sorted data:

  1. Both sources are sorted by a key attribute (e.g., accountId)
  2. Two iterators walk both collections simultaneously
  3. When keys match, each specified attribute is compared using Objects.equals()
  4. When keys don't match, the record is classified as onlyOnA or onlyOnB
  5. Results are stored as ComparisonBreak records
ℹ️

This approach is O(n log n) for sorting + O(n) for the merge pass, making it efficient for datasets with millions of records.
